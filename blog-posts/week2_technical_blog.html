~ Overview of Positioning ~

Positioning: Overview

So today, I decided to delve into positioning for html elements. I thought this would be best as I was struggling generally with the subject but I think I have a better grasp on it now.

CSS positioning, generally, allows you to position an element on a website. To help with understanding this concept, its always good to think of individual elements as their on box. From there, you can logically determine how and where you'd like to position those elements relative to the whole page, or within an other element.

Positioning: Static

As you may or may not know, all elements are defaulted to the position of static. This basically means the element is positioned where it would normally be if no stylings were applied to it. You don't typically have to specify this unless you are trying to override a parent attribute.
This may be obvious, but statically positioned elements are not affected by top, bottom, left and right properties.

Positioning: Fixed

Elements with a fixed position will always remain in the same place relative to the browser window. Even if you scroll, the element will still in the fixed location within the browser window (which can be a cool affect if that's what you're going for!).
Its important to remember that fixed elements are removed from the normal flow of the page. Other elements on the page act as if the fixed element doesn't even exist. (How ruuude! - full house, anyone..?)

Positioning: Absolute

Elements that have an absolute positioning applied are positioned relative to their first parent element that has a position other than position:static. However, if there is no parent element, then the element can be positioned anywhere on the page using top, bottom, left or right.
Similar to fixed elements, absolute elements are removed from the normal flow of the page and other elements act as if the fixed element doesn't even exist. (Again, Ruuude - you should know the reference by now)

Positioning: Relative

For relative positioning, you can move elements to a new locations relative to its normal position. You do this by utilizing the following properties: top, bottom, left or right. This is useful in instances, for example, where you want a couple elements to overlap, such as a header overlapping on to an image.
Important Note: elements with a relative position are usually used for containers of absolutely positioned elements. (Write that down)
